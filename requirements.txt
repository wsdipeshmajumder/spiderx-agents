# Root requirements pin for Railway / Railpack autodetect.
#
# Railpack stages the build Docker-style: it copies requirements.txt
# FIRST, runs `pip install`, and only then copies the rest of the
# source. That means a `-r backend/requirements.txt` reference can't
# resolve — backend/ isn't in the build context yet at install time.
#
# So the dep list is inlined here. Keep this file IN SYNC with
# backend/requirements.txt — both should list the same packages.
# (Local dev uses backend/requirements.txt directly; Railway uses
# this one.)

fastapi>=0.115
# Starlette needs python-multipart to parse `multipart/form-data` bodies. Plain
# `fastapi` (not fastapi[standard]) does NOT pull it in, so without this pin every
# file upload — e.g. knowledge-base .txt/.docx upload — 500s on Railway with
# "Internal Server Error" (works locally only if the venv happens to have it).
python-multipart>=0.0.9
uvicorn[standard]>=0.30
google-genai>=1.40
python-dotenv>=1.0
asyncpg>=0.30
alembic>=1.13
SQLAlchemy>=2.0
psycopg[binary]>=3.2
PyYAML>=6.0
# Python 3.13 removed the stdlib `audioop` module. We import it in
# backend/twilio_bridge.py (mu-law ↔ PCM) and backend/recordings.py
# (16 kHz → 24 kHz resample for the stereo call-recording mix).
# `audioop-lts` is the maintained drop-in backport — same module name,
# same API. Railpack resolves Python 3.13 by default on Railway, so
# this pin is mandatory; on local Python 3.9-3.12 it's a no-op.
audioop-lts>=0.2; python_version >= "3.13"
